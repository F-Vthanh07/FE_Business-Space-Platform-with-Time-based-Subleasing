# Quy trình làm việc (Workflow Rules)

Tài liệu này mô tả chi tiết quy trình phát triển dự án.

## Quy trình Git
- Luôn tạo nhánh mới từ `main` / `develop` trước khi phát triển tính năng.
- Tên nhánh tuân theo format: `feature/tên-tính-năng`, `bugfix/lỗi-cần-sửa`.

## Quy trình Code Review
- Đảm bảo code được test local trước khi tạo Pull Request.
